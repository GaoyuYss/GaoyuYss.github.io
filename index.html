网络相关命令

以太网 –Ethernet
ping -检查网络可达性 - ICMP
tracepath –检查数据路由 （请求的路径通过的路由）
ifconfig 检查网络接口（网络适配器）
ip -查看配置网络接口相关信息
netstat –查看网络状态
netstat -nap n:以数字的形式返回网络状态 ，a -所有的 p -端口
进程，端口，服务起了没有
netstat-nap | grep redis 查找redis服务
sftp -安全网络文件传输
scp -安全文件拷贝
scp 文件 root@1.2.3.4:/目录/文件名 （将文件传到另一名用户）

如何把程序放在后台学习：
redis-server & 在启动redis的时候把程序放在后台运行
redis-server > redis.log & 输出重定向到redis.log

追加输出重定向
2> 错误输出重定向

jobs 查看在后台运行的进程
fg %编号 把后台的运行的进程放在前台运行 （编号是jobs中前面的编号）
bg %编号 让后台停止的程序运行起来
Ctl +z - 将前台进程暂停放在后台

touch 文件名 如果文件的是已经创建的，那么改变他的创建时间，如果不存在就创建一个新的文件

Git的使用
1.创建git仓库
git init

2.将代码纳入暂停区
git add 文件名
git add .

3.查看暂存区状态
git status
git rm –cached 文件名 （将文件从缓存区移除）
git rm 文件名 （删除缓存区，同时工作区也删除文件）

4.提交到本地仓库
git commit -m ‘说明’

5.查看提交日志
git log -查看前面的版本
git reflog -查看到前面和后面的版本

6.回到历史版本和之后的版本
git reset 版本号
git reset –hard HEAD^ 回到上一个版本
git reset –hard HEAD^^ 回到上两个版本
git reset –hard 版本的哈希摘要

–hard参数是为了保持工作区和历史版本的一致性
