django注册、登录系统

##request:向服务器发送的请求
  # COOKIE :传递客户端中的cookie内容

# GET ：获取http GET 请求中传递的参数。如：127.0.0.1:80、xxx/?id=1
# POST :获取http POST请求中传递的参数
# FILES :获取页面中传递的图片文件
# path :获取当前请求的URL路径
# method:获取请求方式,有GET和POST

##HttpResponse：
在django.http模块中定义了HttpResponse对象的API
HttpRequest对象由Django自动创建，HttpResponse对象由程序员创建
不调用模板，直接返回数据

##HttpResponseRedirect:
res = HttpResponseRedirect(‘/my_index/‘)
调用my_index路由，通过路由执行函数

##render:
render(request,’register.html’,{‘err’:err})
render(request, template_name,{‘xxx’, xxx})
结合一个给定的模板和一个给定的上下文字典，并返回一个渲染后的HttpResponse对象
request：该request用于生成response
template_name：要使用的模板的完整名称
context：添加到模板上下文的一个字典，视图将在渲染模板之前调用它
注意：render是将模板渲染到当前路由下，即只是把模板拿过来渲染，并没有到别的路由下，
当使用render得到别的页面时，POST或者GET请求都是给原来的路由处理

##cookie：令牌
  set_cookie(key, value=’’, max_age=None, expires=None)：设置Cookie
key、value都是字符串类型
max_age是一个整数，表示在指定秒数后过期
expires是一个datetime或timedelta对象，会话将在这个指定的日期/时间过期，注意datetime和timedelta值只有在使用PickleSerializer时才可序列化
max_age与expires二选一
如果不指定过期时间，则两个星期后过期

注意：设置筛选后，必须要有令牌才能通过路由到达指定页面，令牌过期也不能通过筛选，
因为数据库中令牌数据过期销毁
