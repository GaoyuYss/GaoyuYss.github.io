模型

一对一: OneToOneField
一对多: Foreignkey
多对多: ManyToManyFiled

#模型中拿到数据的类型

模型名.objects —>拿到路径
模型名.objects.filter(筛选条件) —>列表
模型名.objects.all() —-列表
模型名.objects.filter(筛选条件).all() —>列表
模型名.objects.first() —>对象
模型名.objects.filter(筛选条件).first() ->对象

一对一：
stuinfo.student –>学生对象
stu.stu_info –>信息对象

一对多：
    stu.g                 -->班级对象
    grade.student_set     -->班级对象的路径

多对多：
    stu.c                 -->课程对象的路径
    cou.student           -->学生对象的路径


filter() 、all()   -->可对路径和列表进行筛选
values()           -->可对路径和列表进行展示

first() 、last()   -->可对路径和列表进行选择

总结：路径就相当于原列表，没有进行筛选过，是列表的原始数据

#母板

母版的创建：
    在工程目录下面创建templates文件夹，并在其下面创建三个HTML文件，
    三个文件分别命名为index.html,base.html,base_main.html;
    index.html继承base_main.html，为坑中内容；base_main.html继承base.html，为坑中铺垫条件；
    base.html为母版，具有html文件的结构，负责将坑放在合适的位置

    {% extends 'base.html' %} -->继承base.html文件
    {% extends 'base_main.html' %} -->继承base_main.html文件

解析标签： {% 标签 %} ，标签有for、if、comment、static、ifequal、block、extends
           {% endfor %} ,结束标签（必须要有结束标签）

解析变量： {{ 变量 }}  --> 将{{}}中的变量解析出来

循环：
    forloop.counter     -->顺序编号，从1开始
    forloop.counter0    -->顺序编号，从0开始
    forloop.revcounter  -->逆序编号，到1为止
    forloop.revcounter0 -->逆序编号，到0为止
    forloop.first    -->循环到第一个为Ture，其他都为False
    forloop.last     -->循环到最后一个为Ture,其他都为False

继承：
    {% block xxx %}
			{{block.super}}
		{% endblock %}
    注意：如果没有继承，那么子坑的内容会覆盖父坑的内容

静态加载:
    STATICFILES_DIRS = [
        os.path.join(BASE_DIR, 'static')
    ]

    第一种方式:
    <link href="/static/css/index.css">
    第二种方式:
    {% load static %}
    {% static 'css/index.css' %}
